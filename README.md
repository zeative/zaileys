## Quick start

There is no `await connect()` ŌĆö the client connects on construction, so every handler you register synchronously is wired up before the first event arrives.

```typescript
import { Client } from 'zaileys'

const client = new Client()

client.on('qr', ({ qrString }) => console.log('Scan this QR:', qrString))
client.on('connect', ({ me }) => console.log('Connected as', me.id))

client.on('text', async (msg) => {
  await msg.reply(`You said: ${msg.text}`)
})
```

That is the whole bot. Scan the printed QR via **WhatsApp ŌåÆ Linked Devices**, and every text message gets a reply.

Prefer a pairing code? Provide your number:

```typescript
const client = new Client({ authType: 'pairing', phoneNumber: '6281234567890' })
```

## Why Zaileys

- **Typed events** ŌĆö `on('text' | 'image' | 'reaction' | 'button-click' | 'group-update' | ŌĆ”)` with fully-typed payloads and IntelliSense. No raw Baileys decoding, no `any`.
- **One chainable builder** ŌĆö `client.send(jid).text(ŌĆ”).reply(quoted).mentions([ŌĆ”])` resolves to the sent message key when awaited.
- **Rich & interactive out of the box** ŌĆö native buttons, lists, carousels, and Meta-AI-style rich responses written as plain markdown.
- **Auto lifecycle** ŌĆö QR or pairing-code login, auto-reconnect with backoff, clean logout, optional `ignoreMe`.
- **Pluggable storage** ŌĆö independent `AuthStore` and `MessageStore` interfaces with `file`, `memory`, `sqlite`, `redis`, `postgres`, and `convex` adapters.
- **Batteries included** ŌĆö command framework, broadcast with rate limiting, scheduled sends, and lazy media processing (image/video/audio/sticker).
- **Runs everywhere** ŌĆö dual ESM/CJS with `.d.ts` + `.d.cts` types; verified on Node, Bun, Deno, and Termux.
- **Modern foundation** ŌĆö Baileys `7.0.0-rc13` (includes the CVE-2026-48063 spoofing patch), built and type-checked with the native (Go) TypeScript 7 compiler.

## Install

```bash
npm i zaileys      # or: pnpm add zaileys  ŌĆó  yarn add zaileys  ŌĆó  bun add zaileys
```

Requires **Node.js v20+**. The `file` auth store is the zero-config default and needs nothing else.

> **Termux / Android:** install with `npm install zaileys --legacy-peer-deps`. A plain `npm install` tries to compile `sharp` (a peer dependency of Baileys, no prebuilt ARM binary) and fails; the flag skips it and Zaileys falls back to the bundled `jimp` path. `pnpm`/`yarn` are unaffected.

Other storage backends are **optional peer dependencies** ŌĆö install only the one you use:

```bash
npm i better-sqlite3   # sqlite adapters
npm i redis            # redis adapters
npm i pg               # postgres adapters
npm i convex           # convex adapters
```

`sharp` is an optional accelerator for media/sticker processing; without it Zaileys falls back to a pure-JS path automatically. It is **not** a Zaileys dependency, but Baileys pulls it in as a peer ŌĆö on platforms with no prebuilt binary (Termux/Android, some Alpine) install with `--legacy-peer-deps` to skip it.

## What you can build

### Send anything

```typescript
await client.send(jid).text('Hello there')
await client.send(jid).image('https://example.com/photo.jpg', { caption: 'Nice shot' })
await client.send(jid).poll('Pick one', ['Red', 'Green', 'Blue'])
await client.send(jid).album([
  { type: 'image', src: './a.jpg' },
  { type: 'image', src: './b.jpg' },
])
```

### Interactive messages

Reply, URL, copy, call, reminder, location, and address buttons ŌĆö plus lists and carousels ŌĆö rendered natively on personal accounts.

```typescript
await client.send(jid).buttons(
  [
    { id: 'yes', text: 'Yes' },
    { type: 'url', text: 'Open docs', url: 'https://github.com/zeative/zaileys' },
    { type: 'copy', text: 'Copy code', code: 'ZAILEYS-2026' },
  ],
  { title: 'Pick one', text: 'Tap a button below' },
)

client.on('button-click', (ctx) => console.log('tapped:', ctx.buttonId))
```

### Rich responses, written as markdown

Toggle `{ rich: true }` and write ordinary markdown ŌĆö fenced code (syntax-highlighted), tables, images, and `:::` directives for products, suggestions, and more.

```typescript

  [
    '*Daily brief* ŌśĢ',
    '',
    '```ts',
    "const client = new Client()",
    '```',
    '',
    ':::suggest',
    'See changelog | Upgrade guide',
    ':::',
  ].join('\n'),
  { rich: true, title: '­¤ō░ zaileys' },
)
```

### Commands, broadcast & schedule

```typescript
const client = new Client({ commandPrefix: ['/', '!'] })
client.command('ping', (ctx) => ctx.reply('pong ­¤Åō'))

await client.broadcast(jids, (b) => b.text('Announcement'), { rateLimitPerSec: 5 })
await client.scheduleAt(new Date(Date.now() + 60_000), (b) => b.text('Sends in 1 minute'))
```

### Mutate messages

```typescript

await client.edit(key).text('Edited')
await client.react(key, '­¤æŹ')
await client.delete(key, { forEveryone: true })

## Storage

Auth state and message history use two independent interfaces, so you can mix and match ŌĆö e.g. auth in SQLite, messages in Redis.

```typescript
import { Client, SqliteAuthStore, RedisMessageStore } from 'zaileys'

const client = new Client({
  auth: new SqliteAuthStore({ database: './auth.db' }),
  store: new RedisMessageStore({ url: 'redis://localhost:6379' }),
})
```

| Adapter    | Auth store          | Message store          | Peer dependency  |
| ---------- | ------------------- | ---------------------- | ---------------- |
| `file`     | `FileAuthStore` ŌŁÉ  | ŌĆö                      | none             |
| `memory`   | `MemoryAuthStore`   | `MemoryMessageStore`   | none             |
| `sqlite`   | `SqliteAuthStore`   | `SqliteMessageStore`   | `better-sqlite3` |
| `redis`    | `RedisAuthStore`    | `RedisMessageStore`    | `redis`          |
| `postgres` | `PostgresAuthStore` | `PostgresMessageStore` | `pg`             |
| `convex`   | `ConvexAuthStore`   | `ConvexMessageStore`   | `convex`         |

> ŌŁÉ default. Convex requires deploying the helper functions in [`examples/convex/`](./examples/convex) ŌĆö see that folder's README.

### Bundling & typecheck troubleshooting

- **`bun build` fails with "Browser build cannot import Node.js builtin"** ŌĆö Zaileys (and baileys underneath) is a Node-only library; bun's bundler defaults to a browser target. Pass the target explicitly: `bun build index.ts --target bun` (or `--target node`).
- **`tsc` reports errors inside `node_modules` (`ws`, `thread-stream`, `whatsapp-rust-bridge`)** ŌĆö upstream declaration issues, not yours. Set `"skipLibCheck": true` in your `tsconfig.json`.
