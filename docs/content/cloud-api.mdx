# Official Cloud API

zaileys can run on the **official Meta WhatsApp Cloud API** instead of WhatsApp Web ÔÇö same
client, same builder, same events. Pick the provider at construction time:

```ts
import { Client } from 'zaileys'

const wa = new Client({
  provider: 'cloud',
  cloud: {
    accessToken: process.env.WA_TOKEN!,      // permanent / system-user token
    phoneNumberId: process.env.WA_PHONE_ID!, // sender phone-number id (not the number itself)
    verifyToken: process.env.WA_VERIFY!,     // webhook GET challenge
    appSecret: process.env.WA_APP_SECRET!,   // enables webhook signature verification
    apiVersion: 'v23.0',                     // optional, pinned default
  },
})

wa.on('text', (m) => m.reply(`echo: ${m.text}`))
await wa.send('628xxx').text('halo')
```

No QR, no pairing, no session files ÔÇö the token authenticates you. `connect()` is a
health-check against the Graph API and resolves immediately.

## Webhook

Inbound traffic arrives via Meta webhooks. `wa.webhook()` returns a framework-agnostic
`(req: Request) => Promise<Response>` that handles the GET verification challenge and
verifies the `X-Hub-Signature-256` HMAC on POSTs (when `appSecret` is set):

```ts
// hono
app.all('/webhook', (c) => wa.webhook()(c.req.raw))

// next.js app router (app/api/whatsapp/route.ts)
export const GET = wa.webhook()
export const POST = wa.webhook()
```

See `examples/cloud-express.ts`, `examples/cloud-hono.ts`, `examples/cloud-next-route.ts`.
In the Meta app dashboard, subscribe the webhook to the `messages` field.

## What works on cloud

| Surface | Status |
| --- | --- |
| `send().text/image/video/audio/document/sticker/location/contact` | Ôťů |
| `send().buttons()` (max 3 reply buttons) / `send().list()` | Ôťů |
| `reply()`, `react()`, `forward()` | Ôťů |
| `sendTemplate(to, name, lang, components?)` | Ôťů cloud-only |
| `markRead(messageId, { typing })` | Ôťů cloud-only |
| `downloadMedia(key)` | Ôťů |
| events: `message`, `text`, `image`, ÔÇŽ, `reaction`, `button-click`, `list-select` | Ôťů |
| `message-status` (`sent`/`delivered`/`read`/`failed`) | Ôťů cloud-only |
| commands (`commandPrefix`), middleware, plugins, `broadcast()`, `scheduleAt()` | Ôťů |
| `group`, `community`, `newsletter`, `privacy`, `presence`, `chat`, `contact`, `business`, `profile` | ÔŁî throws `ZaileysProviderError` (`UNSUPPORTED_ON_CLOUD`) |
| `edit`, `delete`, `pin`, `setDisappearing`, stories/status | ÔŁî not offered by the Cloud API |
| QR / pairing / multi-device session | ÔÇö not applicable |

Everything unsupported throws a **typed error** immediately ÔÇö nothing fails silently.

## Management surface ÔÇö `wa.cloud.*`

Business tooling lives under the cloud-only `wa.cloud` namespace. WABA-scoped calls need
`cloud.wabaId`.

```ts
// templates (CRUD + review lifecycle)
await wa.cloud.templates.list({ status: 'APPROVED' })
await wa.cloud.templates.create({ name, category: 'MARKETING', language: 'id', components })
await wa.cloud.templates.delete('promo_juli')
wa.on('template-status', (t) => console.log(t.name, t.event)) // APPROVED/REJECTED/PAUSED

// business profile
await wa.cloud.profile.get()
await wa.cloud.profile.update({ about: 'buka 9-5', websites: ['https://ÔÇŽ'] })

// whatsapp flows
await wa.cloud.flows.list()
await wa.cloud.flows.send(to, { flowId, cta: 'Isi form', bodyText: 'ÔÇŽ', screen: 'BOOKING' })
wa.on('flow-response', (f) => console.log(f.response)) // parsed nfm_reply JSON

// catalog / commerce
await wa.cloud.commerce.sendProduct(to, { catalogId, retailerId })
await wa.cloud.commerce.sendProductList(to, { catalogId, headerText, bodyText, sections })
wa.on('order', (o) => console.log(o.items)) // customer placed an order

// misc
await wa.cloud.sendAddressRequest(to, { bodyText: 'Alamat kirim?', countryIso: 'ID' })
await wa.cloud.blocklist.add(['628xxx'])
await wa.cloud.qr.create('halo!')
await wa.cloud.analytics.conversations({ start, end })
await wa.cloud.phone.requestCode('SMS', 'id')
```

**Campaign recipe**: `wa.cloud.templates.create()` Ôćĺ wait for `template-status` APPROVED Ôćĺ
`broadcast()`/loop `sendTemplate()` Ôćĺ track `message-status` + `wa.cloud.analytics`.

## Notes & ceilings

- Business-initiated conversations require an **approved template** (`sendTemplate`) once
  outside the 24h customer-service window ÔÇö plain sends return error 131047.
- Interactive sends return a builder-generated key id; the real `wamid` is recorded in the
  message store (and referenced by `message-status` events).
- Media headers on interactive buttons are not supported on cloud yet.
- zaileys still imports baileys at module load even in cloud mode (startup cost only, no
  runtime use); lazy loading is a planned optimization.
