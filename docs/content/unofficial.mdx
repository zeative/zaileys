# Unofficial ¬∑ WhatsApp Web

The **default** provider. It automates a WhatsApp Web session through
[Baileys](https://github.com/WhiskeySockets/Baileys) ‚Äî the same multi-device protocol your phone
uses when you link a device. No Meta approval, no business verification: scan a QR (or enter a
pairing code) and you're live in seconds, with the **full power of a personal account**.

```typescript
import { Client } from 'zaileys'

const client = new Client() // provider: 'baileys' is the default

client.on('qr', ({ qrString }) => console.log('Scan this QR:', qrString))
client.on('connect', ({ me }) => console.log('Connected as', me.id))
client.on('text', (msg) => msg.reply(`You said: ${msg.text}`))
```

There is nothing extra to configure ‚Äî every other page in these docs (Events, Sending Messages,
Interactive, Groups, Storage‚Ä¶) describes this provider unless a page is explicitly marked

## Why pick the unofficial provider

- **Zero approval, instant start** ‚Äî no Meta Business account, no WABA, no template review. Great
  for prototypes, personal automation, and community bots.
- **Personal-account superpowers** ‚Äî groups, communities, newsletters (channels), polls, stickers,
  presence, edit, delete, disappearing messages, status. None of these exist on the Cloud API.
- **Message anyone** ‚Äî no 24-hour window, no template gate. You can DM any number that has WhatsApp.
- **Self-contained** ‚Äî a persistent socket, no public webhook URL required. Runs on a Raspberry Pi,
  Termux, a VPS, anywhere Node runs.

## Logging in

<Tabs items={['QR code', 'Pairing code']}>
<Tabs.Tab>
```typescript
const client = new Client()
client.on('qr', ({ qrString }) => console.log(qrString))
```
Scan from **WhatsApp ‚Üí Linked Devices ‚Üí Link a Device**. The session is saved to disk (the `file`
auth store) so you only scan once.
</Tabs.Tab>
<Tabs.Tab>
```typescript
const client = new Client({ authType: 'pairing', phoneNumber: '6281234567890' })
client.on('pairing-code', ({ code }) => console.log('Enter on phone:', code))
```
Enter the code under **Linked Devices ‚Üí Link with phone number instead**. Handy on headless servers
where you can't render a QR.
</Tabs.Tab>
</Tabs>

Sessions persist through the [storage adapters](/storage) ‚Äî `file` (default), `sqlite`, `redis`,
`postgres`, or `convex`. Auto-reconnect with backoff and clean logout are handled for you; see
[Client & Lifecycle](/client).

## What's exclusive to this provider

These features simply don't exist on the official Cloud API ‚Äî the unofficial provider is the only
way to use them in Zaileys:

- [Groups](/groups) ‚Äî create, manage participants, admin actions, invite links.
- [Communities](/community) and [Newsletters / Channels](/newsletter).
- [Presence](/presence) ‚Äî subscribe to others' online/typing state.
- Polls, carousels, stickers, edit, delete, pin, disappearing messages ‚Äî see
  [Sending Messages](/sending-messages) and [Interactive](/interactive).
- [Privacy & Blocking](/privacy) settings and status/stories.

## Know the trade-offs

<Callout type="warning">
This provider is **not endorsed by Meta**. It drives WhatsApp Web, so the number carries a real risk
of temporary or permanent bans ‚Äî especially with bulk or cold messaging. Use a number you can afford
to lose, keep volumes human, and never spam. For a compliant, unbannable business channel with
templates and campaigns, use the **[Official Cloud API](/cloud-api)** instead ‚Äî same code, different
`provider`.
</Callout>
